# 🛡️ Security Policy: wiki-en

> **Part of the [Open Research Collection by Juan Moisés de la Serna Tuya](https://doi.org/10.5281/zenodo.19415092)**

We take the security of our research data and documentation seriously. This policy outlines how to report vulnerabilities and our commitment to maintaining a secure open science environment.

---

## 1. Reporting a Vulnerability

If you identify a security vulnerability within this repository (e.g., exposed credentials, data leaks, or malicious code injection), please **do not** open a public issue. Instead:

1.  **Email:** Contact the lead researcher at [juanmoises.delaserna@unir.net](mailto:juanmoises.delaserna@unir.net) with the subject line `SECURITY VULNERABILITY REPORT`.
2.  **Details:** Provide a detailed description of the vulnerability, steps to reproduce it, and any potential impact on the research data.
3.  **Response Time:** We will acknowledge your report within **48 hours** and provide a timeline for resolution.

---

## 2. Supported Versions

We only provide security updates for the **main** branch of this repository.

| Version | Supported          |
| ------- | ------------------ |
| latest  | ✅ Yes             |
| < 1.0   | ❌ No              |

---

## 3. Data Integrity & PII

This repository contains research documentation and metadata. We are committed to ensuring that:
- No **Personally Identifiable Information (PII)** is ever included in our datasets.
- All data is anonymized or derived from public sources.
- Our build processes (Jekyll/GitHub Actions) follow standard security best practices.
